# Alibaba Inquiry Auto-Reply System

> Alibaba 国际站询盘全自动检测 + AI 智能回复系统

## How It Works

```
Buyer sends inquiry
    │
    ▼ ~1 second
Chrome Extension detects unread (MutationObserver)
    │
    ▼ ~2 seconds
Bridge Server writes to pending queue
    │
    ▼ < 5 minutes
Cron triggers AI processing
    │
    ▼ ~30 seconds
AI reads inquiry → generates reply → sends via browser
```

## Quick Start

### 1. Install Chrome Extension
```
chrome://extensions/ → Developer mode → Load unpacked → select alibaba-inquiry-monitor/
```
Keep https://message.alibaba.com/message/default.htm#feedback/all open in a tab.

### 2. Start Bridge Server
```bash
cd alibaba/
python bridge_server.py
```
You should see: `[Bridge] Listening on http://127.0.0.1:9876`

### 3. Configure Cron (in Accio Work)
Ask your Accio AI agent:
> "Set up a cron job to check pending_inquiries.json every 5 minutes and auto-reply to inquiries"

## Requirements

- **Chrome** with Alibaba.com logged in
- **Python 3** for bridge server
- **Accio Work** with Alibaba Chat & Analysis SubAgent
- **Accio Browser Relay** enabled

## Each File - Detailed

| File | Who Runs It | What It Does |
|------|------------|--------------|
| `content.js` | Chrome (on message.alibaba.com) | Watches the page DOM for unread inquiries using MutationObserver. Reports every unread inquiry every 3 seconds. |
| `background.js` | Chrome (service worker) | Receives inquiry data from content script, forwards it via HTTP POST to bridge server (port 9876). Handles notification clicks. |
| `popup.html/js` | Chrome (extension popup) | Control panel. Toggle on/off, set active hours (00-00 = all day, 17-08 = night only), view activity log. |
| `manifest.json` | Chrome | Declares extension permissions (storage, alarms, notifications, localhost access). |
| `bridge_server.py` | Your PC (terminal) | Python HTTP server. Listens for POST from extension, writes inquiry data to pending_inquiries.json. Deduplicates entries. |
| `pending_inquiries.json` | Auto-generated | JSON array of inquiries waiting for AI reply. Bridge writes, cron reads and clears. |
| `known_inquiries.json` | Auto-generated | JSON array of inquiry IDs already replied to. Cron adds after each successful reply. Prevents re-replying to handled conversations (but allows re-queue if buyer sends new message). |

## Permission Notes

- **notifications**: Desktop popup when new inquiry detected
- **storage**: Save settings (time range, enable/disable state)
- **alarms**: Keep service worker alive
- **http://127.0.0.1:9876/**: Bridge server communication

## Reusing for Other Stores

1. Install extension (same extension, no changes needed)
2. Start bridge server (same)
3. Update cron's AI message to use your store name
4. Ensure Alibaba account is logged in Chrome
